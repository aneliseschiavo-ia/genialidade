**Status:** 🆕 Draft

**Assigned to:** @dev + You (copywriting review)
**Depends on:** Story 5.1 (Landing page), Story 5.2 (Email sequences)

---

## 📋 Descrição

Integrar e sincronizar **copy em todos touchpoints** (landing page, emails, portal, etc):

1. **Mapping de copy:**
   - Landing page (8 seções) → Story 5.1
   - Email sequences (5 emails) → Story 5.2
   - Portal cliente (headlines, msgs) → Story 1.6
   - CTA buttons (variações) → Stories múltiplas
   - Social proof (depoimentos) → Story 5.1 + banco de dados

2. **Content management:**
   - Centralizar copy em `docs/copywriting/` (não hardcoded em componentes)
   - Versionamento (v1, v2 se A/B test depois)
   - Fácil atualização (não precisa dev para mudar headlines)

3. **A/B testing prep:**
   - Setup para testar variações de headlines
   - Setup para testar variações de CTA copy
   - Tracking de qual versão cliente vê

4. **Refinement:**
   - Você revisa copy atual (`landing-page-formulario.md`)
   - Sugere refinamentos baseado em feedback inicial
   - Testa com clientes piloto (Luiz, Bruna, Daniele, Gustavo, Nina)
   - Itera v1 → v2 → v3 conforme aprende

## 📌 Notas

- **YAML é simples:** Você consegue editar sem ser dev. Mantém versionamento + histórico
- **Não hardcode:** Depois fica fácil A/B test. Só muda YAML, não precisa redeploiar
- **Variações de agora:** Já tem 4 CTAs em `landing-page-formulario.md` — coloca todas em YAML
- **A/B depois:** Não precisa rodar teste agora, mas setup permite fazer sem dev
- **Integração é contínua:** Conforme aprende com clientes piloto, atualiza copy (v1 → v2)
- **Social proof:** Depois que tiver 3 clientes reais, coloca depoimentos em banco (mais escalável)
