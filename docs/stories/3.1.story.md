## 📋 Descrição

Criar **documento estruturado + checklist** que você usa DURANTE a sessão 1:1 para:

1. **Facilitação eficiente:** Não esquecer nada importante em 90 min
2. **Extração de insights:** Questões-chave que aprofundam entendimento
3. **Documentação em tempo real:** Campos para preencher durante sessão
4. **Geração de entregáveis:** Dados coletados viram Blueprint 7D (Story 3.3)
5. **Transição suave:** De "hipótese preliminar" (Fase 1) para "diagnóstico validado" (Fase 2)

Será documento estruturado (PDF ou Google Doc) que você abre no início da sessão.

---

## ✅ Acceptance Criteria (10 pontos)

- [ ] 1. **Agenda estruturada:** 90 min divididos em 7 seções (15min aquecimento, 40min aprofundamento, 20min síntese, 15min Blueprint)
- [ ] 2. **Questões-chave:** 15-20 perguntas abertas que refinam hipótese preliminar (baseadas em 4 dimensões: Desalinhamento, Ruído, Vazamento, Maturidade)
- [ ] 3. **Framework extração:** Método claro para ir de respostas → Verdade Núcleo refinada (ex: buscar padrões, conectar pontos)
- [ ] 4. **Campos documentação:** Seções para preencher em tempo real (Observações, Padrões Detectados, Cortes Propostos, Decisões, Indicador)
- [ ] 5. **Verdade Núcleo refinada:** Template para escrever hipótese refinada em 1-2 frases claras (espaço para iteração)
- [ ] 6. **3 Cortes identificados:** Campos estruturados (O quê, Por quê, Impacto esperado) para cada corte
- [ ] 7. **3 Decisões definidas:** Campos estruturados (Decisão, Métrica, Responsável, Timeline) para cada decisão
- [ ] 8. **Plano 7D sketch:** Template simples (Dia 1: X, Dia 2: Y, ... Dia 7: teste) para esboçar blueprint
- [ ] 9. **Indicador crítico:** Campo para definir métrica que validará se premissa está certa (ex: "receita por hora", "custo por atividade")
- [ ] 10. **Guia facilitação:** Dicas práticas (como fazer perguntas abertas, reconhecer resistência, sintetizar, transição para próximos passos)

## 📌 Notas

- **Você vai usar isso ao vivo:** Imprimir ou abrir em tablet durante sessão
- **Não é rígido:** É guia flexível. Se cliente vai em direção diferente, siga momentum
- **Documentação em tempo real:** Toma notas enquanto fala (depois Story 3.3 transcreve em PDF formal)
- **Verdade Núcleo é iterativa:** Começa com hipótese Fase 1, refina na sessão, sai com versão final
- **Cortes e Decisões devem ser específicos:** Não genéricos (ex: "elimina reúnião status 2x/semana de 1h cada = 8h/semana liberadas")
- **Indicador é tudo:** Se cliente não sabe medir se premissa funciona, sessão falhou
