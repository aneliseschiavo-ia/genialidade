## 📋 Descrição

Construir **formulário web interativo com 28 questões** que:
1. **Comunica valor** (pitch silencioso — cada pergunta reforça que você entende o problema cliente)
2. **Coleta dados estruturados** para diagnóstico (Desalinhamento, Ruído, Vazamento, Maturidade)
3. **Guia cliente intuitivamente** (não é genérico, é bem específico)
4. **Persiste respostas** em banco de dados (Story 1.1)
5. **Mostra progresso visual** (X de 28 questões, barra de progresso)

Referência: `landing-page-formulario.md` → Seção "O Formulário Como Pitch" + Bloco 1-6.

## 📌 Notas

- 28 questões estruturadas em **6 blocos temáticos** (ver `landing-page-formulario.md`):
  - Bloco 1 (Q1-5): Estrutura — "Entendo seu faturamento e operação"
  - Bloco 2 (Q6-10): Prioridade — "Entendo seu caos de decisões"
  - Bloco 3 (Q11-15): Coerência — "Entendo seu desalinhamento"
  - Bloco 4 (Q16-20): Financeiro — "Entendo seu vazamento"
  - Bloco 5 (Q21-25): Padrão — "Entendo seu loop repetido"
  - Bloco 6 (Q26-28): Maturidade — "Entendo se você está pronto pra mudança"

- Cada questão tem um **propósito funcional** (coleta dados) e **propósito de pitch** (comunica valor)
- Autossalva é critical: cliente não pode perder resposta depois de 20 questões
- Layout não deve ser "formulário genérico" — deve ser **elegante e específico** (design em Story 5.1)
