# Story 4.1: Dashboard Progresso 90D — Rastreamento Indicador

## 📋 Descrição

Criar **dashboard visual no portal cliente** que rastreia:

1. **Indicador crítico (evolução semanal):**
   - Valor inicial (dia 1)
   - Valor atual (hoje)
   - Meta dia 7
   - Meta dia 90
   - Gráfico de linha (dias 1-90)

2. **Status plano 7D:**
   - Check-in enviado? ✅/❌
   - Feedback recebido? ✅/❌
   - Ações executadas? (checklist)

3. **Timeline:**
   - Fase 1 (dias 1-5) — ✅ completo
   - Fase 2 (dia 8) — ✅ sessão feita
   - Plano 7D (dias 8-15) — 📍 in progress
   - Check-ins semanais (dias 15-90) — 📍 in progress
   - Sessões mensais (dias 15, 36, 57, 78) — 📍 upcoming

4. **Notificações:**
   - "Você completou 4/5 dias do plano 7D"
   - "Check-in da semana 2 está atrasado — enviar agora?"
   - "Próxima sessão: terça 14h, preparar com pontos chave"

---

## ✅ Acceptance Criteria (10 pontos)

- [ ] 1. **Gráfico indicador:** Linha temporal (eixo X = dias 1-90, Y = valor indicador) atualiza em tempo real
- [ ] 2. **Valores referência:** Mostra inicial (dia 1), atual (hoje), meta 7D, meta 90D (claramente marcados no gráfico)
- [ ] 3. **Status Plano 7D:** 3 checkboxes (check-in enviado, feedback recebido, ações executadas) atualizam baseado em dados
- [ ] 4. **Timeline visual:** 4 fases (Fase 1, Fase 2, Plano 7D, Check-ins) com barra de progresso por fase
- [ ] 5. **Próximas ações:** "Próxima sessão em X dias" + link para agendar
- [ ] 6. **Notificações:** Sistema simples (não push, só visual no dashboard) alertando atraso check-in ou próxima sessão
- [ ] 7. **Mobile responsivo:** Dashboard é acessível em celular (gráfico não fica muito pequeno)
- [ ] 8. **Dados em tempo real:** Se cliente preenche check-in, aparece no dashboard em <1 segundo
- [ ] 9. **Histórico:** Cliente pode ver semanas anteriores (semana 1, 2, etc) — dropdown ou visualização anterior
- [ ] 10. **Performance:** Dashboard carrega em <2 segundos mesmo com 90 dias de dados

## 📌 Notas

- **Não é app complicado:** Dashboard é simples, visual, sem muitos cliques. Cliente vê progresso, é suficiente
- **Gráfico é motivacional:** Ver linha subindo (ou não) é impactante. Reforça valor do acompanhamento
- **Notificações não são push:** Só visual no dashboard (não spam email/SMS)
- **Dados vêm de Story 4.2:** Dashboard é consumer, Story 4.2 é produtor (check-in data)
- **Histórico é importante:** Cliente quer revisar "como estava semana 3?" — dropdown para semanas anteriores
