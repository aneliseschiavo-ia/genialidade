## 📋 Descrição

Implementar **sistema automático de email** que:

1. **Trigger:** Quando score (Story 1.3) é calculado e persistido no banco
2. **Template personalizado:** Envia email com:
   - **Mensagem de recebimento** ("Recebemos seu formulário...")
   - **Score detalhado** (4 dimensões de 0-5, total 0-20)
   - **Hipótese preliminar** (3-5 frases específicas ao cliente)
   - **Decisão clara** ("✅ APROVADO — você foi selecionado" ou "⏸ REJEITADO — recomendamos Y antes")
   - **Próximas etapas** (se aprovado: "Você receberá convite para sessão em 24h", se rejeitado: "Conectamos em [data]")
3. **Timing:** Envia automaticamente 24-48h após formulário (não instantâneo)
4. **Rastreamento:** Log de envio em banco de dados para auditoria
5. **Graceful error handling:** Se email falha, não quebra fluxo (retry automático, notifica admin)

## 📌 Notas

- **Email é operação crítica:** Cliente precisa receber score para progredir para Fase 2
- **Timing não é instantâneo:** 24-48h é intencional (dá tempo para você revisar padrões antes de disparar email)
- **Graceful failure:** Se SendGrid falha, não quebra fluxo — job retenta 2x, então notifica você (admin email)
- **Personalization é key:** Email "genérico" reduz conversão Fase 1→2. Hipótese específica reforça que você entende o problema
- **Preview text matters:** "Seu diagnóstico saiu!" vs "Nova notificação" — primeira ganha 30% mais opens
- **Unsubscribe graceful:** Não proibir unsubscribe (GDPR), mas capture motivo antes de permitir
