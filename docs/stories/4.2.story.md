## 📋 Descrição

Criar **template reutilizável de 5 campos** que cliente preenche **toda semana** (13 vezes, dias 8-90):

1. **Campo 1: Indicador da Semana**
   - "Qual foi o valor do seu indicador critíco essa semana?" (número, ex: receita/hora = 250)
   - Obrigatório

2. **Campo 2: Ações Executadas**
   - "Você executou as 3 ações do plano? (sim/não/parcialmente)"
   - Checklist com 3 itens (da Story 3.3)
   - Obrigatório

3. **Campo 3: Bloqueadores/Dificuldades**
   - "O que foi mais difícil essa semana?" (texto livre, 200 caracteres)
   - Opcional mas recomendado

4. **Campo 4: Aprendizados**
   - "1 coisa que você aprendeu que pode escalar?" (texto livre, 200 caracteres)
   - Opcional mas recomendado

5. **Campo 5: Próxima Semana**
   - "Qual é o foco principal para próxima semana?" (texto livre, 100 caracteres)
   - Obrigatório (você depende disso para feedback)

**Armazenamento:**
- Cada check-in salvo no banco (Story 1.1 — tabela `check_ins`)
- Data/hora automática
- Visível para você (admin) e cliente (no Caderno, Story 3.2)

**Automação:**
- Email de lembrete segunda-feira (você quer que cliente envie até domingo?)
- Ou notificação no portal
- 1 email por semana máximo (não spam)

## ✅ Acceptance Criteria (10 pontos)

- [ ] 1. **5 campos exatos:** Indicador, Ações, Bloqueadores, Aprendizados, Próxima Semana (estrutura rígida)
- [ ] 2. **Indicador numérico:** Campo aceita apenas número (float), com min/max se houver (ex: não pode ser negativo)
- [ ] 3. **Ações checklist:** 3 checkboxes da semana (puxa de Story 3.3 Blueprint)
- [ ] 4. **Campos texto:** Bloqueadores + Aprendizados + Próxima com limite caracteres (200, 200, 100)
- [ ] 5. **Validação:** Indicador + Ações + Próxima obrigatórios; Bloqueadores + Aprendizados opcionais
- [ ] 6. **Autossalva:** Check-in salvo em tempo real (não precisa botão "salvar")
- [ ] 7. **Data/hora:** Timestamp automático, mostra quando foi preenchido
- [ ] 8. **Histórico semanas:** Cliente pode revisar check-ins anteriores (dropdown ou tabela com semanas 1-13)
- [ ] 9. **Lembrete automático:** Email/notificação 1x por semana (segunda-feira?) lembrando preencher
- [ ] 10. **Persistência banco:** Check-in salvo em tabela `check_ins` (cliente_id, week_number, campos, timestamp)

## 📌 Notas

- **5 campos é máximo:** Não adicione mais (cliente já está ocupado executando plano). Simples é melhor
- **Indicador é métrica crítica:** Esse número é tudo que importa. Tudo mais é contexto
- **Ações vêm de Story 3.3:** Template puxa 3 ações do Blueprint cliente. Cliente marca se fez
- **Aprendizados é gold:** "1 coisa que você aprendeu" é dado valioso para feedback (Story 4.3)
- **Próxima semana é planejamento:** Você depende disso para saber foco cliente semana seguinte (Story 4.3 feedback)
- **Lembrete é importante:** Conformidade métrica #5 é ≥90% enviam. Lembrete aumenta taxa
- **Opcional é respeitoso:** Bloqueadores e Aprendizados opcionais, cliente não se sente sobrecarregado
