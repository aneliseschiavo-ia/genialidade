## 📋 Descrição

Criar **email automático de feedback** que você envia **24h após cliente preencher check-in**:

1. **Trigger:** Story 4.2 (cliente preenche check-in)
2. **Delay:** Aguarda 24h para você tomar ação/revisar
3. **Conteúdo (4 seções):**
   - **Validação:** Você confirma que recebeu + calcula progresso indicador
   - **Padrões:** Extrai padrões (ex: "bloqueador recorrente é XYZ")
   - **Recomendação:** Baseada em bloqueador + aprendizado + próxima semana, sugere ajuste
   - **Próxima Semana:** Recap foco que cliente disse + ajuste sua recomendação

4. **Automação:**
   - Job que roda 24h após check-in
   - Você define template + lógica feedback (ou é totalmente automático?)
   - Email enviado cliente + cópia para você

5. **Tone:** Validador + coach (celebra progresso, reconhece dificuldades, orienta próximos passos)

## ✅ Acceptance Criteria (10 pontos)

- [ ] 1. **Trigger 24h:** Job processa check-in, aguarda 24h, depois envia feedback email
- [ ] 2. **Seção Validação:** Mostra indicador semana anterior vs atual, calcula % mudança (ex: "cresceu 5%")
- [ ] 3. **Seção Padrões:** Analisa bloqueadores últimas 2-4 semanas, identifica padrão (ex: "reuniões estão drenando seu tempo")
- [ ] 4. **Seção Recomendação:** Baseada em bloqueador + aprendizado, sugere 1 ação concreta (ex: "reduz reuniões de 3x para 1x/semana")
- [ ] 5. **Seção Próxima Semana:** Recap "você disse foco é X, nós recomendamos Y" — alinhamento claro
- [ ] 6. **Template personalizado:** Email menciona nome cliente, indicador específico, bloqueador específico (não genérico)
- [ ] 7. **Email log:** Registrado em `email_log` (tipo="weekly_feedback", cliente_id, timestamp, content_version)
- [ ] 8. **Cópia admin:** Você recebe cópia com mesmo feedback (aprova automático ou manual?)
- [ ] 9. **Móvel responsivo:** Email legível em celular
- [ ] 10. **Teste E2E:** Cliente preenche check-in → 24h depois job roda → email enviado cliente + você recebe

## 📌 Notas

- **Feedback é relacional:** Não é robô comentando. É você (personificado) reconhecendo esforço + orientando próximos passos
- **Padrões identificam raízes:** Se bloqueador aparece 2-3x, é sintoma de problema estrutural (oportunidade upsell)
- **Recomendação é coaching:** Não mandatório, é "sugestão baseada em seu feedback". Cliente decide se aceita
- **Timing é crítico:** 24h é janela ótima (cliente ainda está pensando na semana, não esqueceu contexto)
- **Você automático ou manual?** Questão importante: feedback é 100% automático (gerado script) ou você revisa antes enviar?
  - Automático = escala melhor
  - Manual = mais qualidade mas tempo seu
  - Recomendo hybrid: automático pero gerar draft, você revisa 5min antes enviar
- **Padrão é ouro:** Se vir padrão bloqueador, é hipótese para conversa mensal (Story 4.4)
