# Story 4.5: Relatório ROI Final — E9 Entregável (Dia 90)

**EPIC:** 4 — Acompanhamento 90D
**Story ID:** 4.5
**Status:** 🆕 Draft
**Priority:** 🔴 MUST (E9 Entregável obrigatório)
**Assigned to:** @dev

---

## 📋 Descrição

Gerar **relatório formal (PDF + email)** no **dia 90 exatamente** que:

1. **Conteúdo (5 seções):**
   - **P1: Evolução Indicador** — Dia 1 vs Dia 90, % mudança, gráfico evolução
   - **P2: Ações Executadas** — O que cliente fez (cortes, decisões, pivots)
   - **P3: ROI Calculado** — Fórmula: (Receita gerada - Investimento) / Investimento
   - **P4: Aprendizados** — Top 3 insights de 13 semanas
   - **P5: Próximos Passos** — Recomendações para continuar escalando (base para renovação 90D ou escalação)

2. **Automação:**
   - Job roda dia 90 exatamente
   - Coleta dados: Indicador (Story 4.1), notas sessões (Story 4.4), Blueprint original (Story 3.3)
   - Gera PDF, armazena, envia email

3. **Timing:**
   - Enviado dia 90 à noite (cliente já tinha 90 dias completos)
   - Momentos chave: validação final, celebração, transição para próxima fase (renovação ou escalação)

4. **Follow-up:**
   - Email com relatório + link para agendar chamada "próximos passos" (renovação 90D outro ciclo, ou escalação novo programa)

---

## ✅ Acceptance Criteria (10 pontos)

- [ ] 1. **P1 Indicador:** Tabela (Dia 1, Dia 7, Dia 30, Dia 60, Dia 90) + % mudança total + gráfico linha
- [ ] 2. **P2 Ações Executadas:** Recap ações cliente fez (de Blueprint Story 3.3 + notas sessões 4.4)
- [ ] 3. **P3 ROI:** Fórmula clara, números concretos (ex: "Receita gerada: R$15k, Investimento: R$2.5k, ROI: 500%")
- [ ] 4. **P4 Aprendizados:** Top 3 insights extraídos de 13 check-ins + 4 sessões (não genéricos, específicos ao cliente)
- [ ] 5. **P5 Próximos Passos:** 3 recomendações baseadas em trajetória (ex: "consolidar nova operação", "escalar para 2 ofertas", "contratar executor")
- [ ] 6. **Timing automático:** Job dispara dia 90 exatamente (não dia 91 ou 89)
- [ ] 7. **PDF branding:** Cores genialidade, layout profissional, assinatura digital você
- [ ] 8. **Email envio:** Relatório anexado + copy celebrando resultado + CTA próxima fase (renovação ou escalação)
- [ ] 9. **Armazenamento:** PDF salvo em Storage, versionado (cliente pode acessar sempre)
- [ ] 10. **Teste E2E:** Job roda dia 90 → PDF gerado com dados corretos → email enviado cliente → você recebe cópia

## 📌 Notas

- **ROI é validação:** Métrica #3 (`refinements-moscow-metricas.md`) pede ≥3x ROI para ≥80% clientes. Relatório prova isso

- **Próximos passos:** Recomendações não são mandatório, mas preparam cliente para decisão (renovação 90D novo ciclo vs escalação)

- **PDF armazenado:** Cliente tem acesso permanente ao relatório (legal + motivacional para voltar)
