## 📋 Descrição

Implementar **sistema automático de scoring** que:

1. **Recebe respostas** do formulário (Story 1.2)
2. **Calcula 4 dimensões** independentes:
   - **Desalinhamento** (Q1-5): Mede desvio entre estratégia declarada e realidade operacional
   - **Ruído** (Q6-10): Mede quantidade de prioridades conflitantes/mudanças de rota
   - **Vazamento** (Q16-20): Mede drenagem financeira (tempo desperdiçado, margens baixas)
   - **Maturidade** (Q26-28): Mede disposição/readiness para mudança estrutural
3. **Gera score total** (0-20, cada dimensão 0-5)
4. **Aprova/rejeita cliente** baseado em lógica:
   - ✅ **APROVADO:** Score ≥12 AND Maturidade ≥4
   - ❌ **REJEITADO:** Score <12 OR Maturidade <4 (feedback estruturado, reconectar depois)
5. **Gera hipótese preliminar** (estrutura de 3-5 frases apontando problema raiz)
6. **Persiste score + decisão** em banco (Story 1.1 — tabela `decisoes`)

## 📌 Notas

- **Scoring é determinístico:** Mesmas respostas = sempre mesmo score (importante para auditoria)
- **Hipótese preliminar não é diagnóstico final** — é uma "pista" para conversa na Fase 2 (Story 2.1)
- **Edge cases críticos:**
  - Score exatamente 12 → APROVADO (≥ incluso)
  - Maturidade exatamente 4 → APROVADO (≥ incluso)
  - Score 11.9 → REJEITADO (não arredonda para 12)
- **Rejeição graceful:** Se rejeitado, feedback deve ser estruturado (não dizer "você não qualifica") mas "essa abordagem requer X antes de começar"
- **Timing:** Scoring deve rodar automaticamente 24-48h após formulário enviado (não instantâneo, dá tempo para você analisar padrões)
