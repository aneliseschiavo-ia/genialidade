## 📋 Descrição

Criar **documento digital compartilhado (Google Docs/Notion)** que:

1. **Começa na sessão:** Você + cliente criam juntos durante Fase 2 (Story 3.1)
2. **Vive por 90D:** Cliente tem acesso exclusivo por 90 dias de acompanhamento (EPIC 4)
3. **Editável em tempo real:** Você escreve → cliente vê mudanças ao vivo
4. **Histórico rastreável:** Sistema de versões, você vê quem mudou o quê e quando
5. **Estrutura clara:** 7 seções (Verdade Núcleo, Premissa Corrigida, Cortes, Decisões, Plano 7D, Indicador, Check-in Template)
6. **Pull from Fase 1:** Puxa contexto de formulário + score + diagnóstico (Story 1.1-1.5)
7. **Integração com Fase 2 & 3:** Check-ins semanais (EPIC 4) atualizam este documento

## 📌 Notas

- **Google Docs é mais simples:** Cliente já conhece, não precisa aprender novo tool. Desvantagem: menos estruturado que Notion
- **Notion é mais poderoso:** Tabelas, databases, automações. Desvantagem: cliente precisa aprender interface
- **Histórico versões é critical:** Você precisa rastrear evolução (semana 1 vs semana 7 do indicador)
- **Expiração automática:** Dia 91 meia-noite, acesso vira read-only (cliente não perde documento, só não pode editar)
- **Não é app:** É documento, deve ser simples para cliente navegar
- **Integração Fase 1:** Puxa info de formulário (respostas específicas), score (4 dimensões), diagnóstico PDF (Verdade Núcleo)
