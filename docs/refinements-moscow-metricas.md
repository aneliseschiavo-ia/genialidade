## Objetivo

Definir escopo com priorização clara (Must/Should/Could/Won't) e refinar métricas de sucesso para ser 100% SMART (Specific, Measurable, Achievable, Relevant, Time-bound).

## 1. PRIORIZAÇÃO MoSCoW — MVP (Diagnóstico + Blueprint 7D)

### **MUST (Não Funciona Sem)**

Essas features são críticas. Se remover uma, a oferta desaba.

```
✅ MUST #1: Formulário 28 Questões (Fase 1)
   - 6 blocos, 28 questões estratégicas
   - Web com login/senha
   - Respostas salvas em banco de dados
   - Obrigatório pra prosseguir

✅ MUST #2: Sistema de Scoring (Fase 1 Resultado)
   - 4 dimensões (Desalinhamento, Ruído, Vazamento, Maturidade)
   - Cálculo automático: 0–5 pontos por dimensão = 0–20 total
   - Threshold claro: Score ≥12 AND Maturidade ≥4 = APROVADO
   - Threshold para rejeição: < critério = REJEITADO com feedback

✅ MUST #3: Sessão 1:1 (Fase 2)
   - 75–90 minutos ao vivo
   - Validação de hipóteses
   - Identificação da premissa distorcida
   - Definição de Verdade Núcleo + 3 Cortes + 3 Decisões
   - Não pode ser assíncrono — DEVE ser conversação real

✅ MUST #4: Blueprint 7D (Entregável Tangível)
   - 7 páginas estruturadas (Verdade Núcleo, Premissa Corrigida, Cortes, Decisões, Plano, Indicador, Check-in)
   - Máx 7 páginas (não pode explodir)
   - Específico ao cliente (não template genérico)
   - Executável (cliente consegue rodar sozinho)

✅ MUST #5: Caderno Inteligente Compartilhado
   - Google Doc ou web app compartilhado
   - Cliente consegue preencher (campos editáveis)
   - Você consegue ver preenchimento (rastreamento)
   - Acesso válido por 90 dias mínimo
   - Não é PDF — DEVE ser interativo
```

---

### **SHOULD (Muito Importante, Mas Pode Ajustar)**

Se remover, a experiência piora, mas a oferta ainda funciona.

```
🟡 SHOULD #1: Diagnóstico Formal em PDF
   - 4–5 páginas consolidadas
   - Entregar dentro de 15 dias (após sessão)
   - Professional design (não Word bruto)
   - Cópia para seu arquivo + cliente

🟡 SHOULD #2: Email Automático de Aprovação/Rejeição
   - Após enviar formulário, cliente recebe confirmação
   - Se aprovado: "Você passou! Próximo: Fase 2 agendada"
   - Se rejeitado: Feedback estruturado + opção para depois

🟡 SHOULD #3: Matriz Visual Resumida (1 página)
   - Resumo gráfico do diagnóstico
   - Cortes visuais, decisões visuais
   - Referência rápida para cliente

🟡 SHOULD #4: Relatório de Análise (Post-Formulário)
   - Você documenta: Dimensões calculadas, hipótese preliminar, decisão
   - Arquivo interno (cliente não vê)
   - Para rastreamento seu
```

---

### **COULD (Legal, Mas Não Crítico)**

Adiciona valor se houver tempo, mas não é bloqueador.

```
💙 COULD #1: Integração com Calendário
   - Agendamento automático de Fase 2 (Calendly, Doodle)
   - Em vez de enviar links manualmente

💙 COULD #2: Relatório Automático de Score
   - Sistema calcula e envia score automaticamente
   - Em vez de você calcular manualmente

💙 COULD #3: Template de Follow-up via Email
   - Sequência automática após formulário (3–5 emails)
   - Lembretes se cliente não preencher

💙 COULD #4: Dashboard de Cliente
   - Cliente vê seu score, hipótese, status do diagnóstico
   - Não é essencial (email já faz)
```

---

### **WON'T (Fora do Escopo MVP)**

Não fazer na primeira versão.

```
🚫 WON'T #1: Integrações com Ferramentas Externas
   - Não integrar com ClickUp, Jira, Notion (MVP 1)
   - Pode fazer depois

🚫 WON'T #2: Análise Automática via IA
   - Não usar ChatGPT/Claude pra gerar diagnóstico
   - MVP é manual (você analisa)

🚫 WON'T #3: Mobile App Nativo
   - Formulário funciona web/mobile responsivo
   - Não precisa app iOS/Android

🚫 WON'T #4: Suporte Post-Blueprint
   - MVP não inclui suporte além de 90 dias
   - Quem quer suporte contrata acompanhamento 90D
```

## 2. PRIORIZAÇÃO MoSCoW — ACOMPANHAMENTO 90D

### **MUST (Não Funciona Sem)**

```
✅ MUST #1: Check-in Semanal (Template)
   - [ ] Executei ações? [ ] % tempo [ ] Indicador [ ] Bloqueios [ ] Vitórias
   - Google Form OU Google Doc OU Airtable
   - Assíncrono (não é reunião)
   - Autossalva

✅ MUST #2: Feedback Semanal (Seu Email)
   - Resposta dentro 24h
   - 4 seções: Validação | Padrões | Recomendação | Próxima semana
   - Profissional + supportivo

✅ MUST #3: 4 Sessões Mensais (Semanas 1, 5, 9, 13)
   - 45 minutos cada
   - Ao vivo (não assíncrono)
   - Agendado com 24h de antecedência
   - Notas documentadas após

✅ MUST #4: Caderno Inteligente Atualizado
   - Cliente preenche check-in lá
   - Você documenta feedback lá
   - Histórico de 13 semanas rastreável
   - Referência clara para cliente
```

---

### **SHOULD (Muito Importante, Mas Pode Ajustar)**

```
🟡 SHOULD #1: Relatório Final de ROI
   - 1–2 páginas consolidadas
   - Indicador inicial vs final
   - ROI documentado
   - Enviado no dia 90

🟡 SHOULD #2: Documentação de Próximos Passos
   - 4 opções claras: Renovar | Escalar | Referências | Encerrar
   - Proposta de preço (se renovar/escalar)
   - Enviado com relatório final
```

---

### **COULD (Legal, Mas Não Crítico)**

```
💙 COULD #1: Gravação das Sessões Mensais
   - Para cliente rever se quiser
   - Armazenagem privada

💙 COULD #2: Template de Escalação Customizado
   - Baseado no ROI do cliente
   - Mostrando próximos investimentos

💙 COULD #3: Comunidade/Grupo de Clientes 90D
   - Slack, Discord, ou WhatsApp
   - Conexão entre clientes em acompanhamento
   - Networking + aprendizado
```

---

### **WON'T (Fora do Escopo MVP 90D)**

```
🚫 WON'T #1: Coaching Contínuo
   - 90D é check-ins + sessões estruturadas
   - Não é "eu disponível 24/7"
   - Quem quer mais contrata novo pacote

🚫 WON'T #2: Treinamento de Equipe
   - MVP é só para o cliente principal
   - Expansão pra equipe é novo pacote

🚫 WON'T #3: Integração com Ferramentas do Cliente
   - Não se integra com CRM, ERP, etc
   - Cliente é responsible por update manual
```

## 3. MÉTRICAS SMART (Específico, Mensurável, Alcançável, Relevante, Temporal)

### **MVP — Diagnóstico (30 dias)**

#### **Métrica #1: Taxa de Conclusão Fase 1→2**

**Antes (SOFT):** "Muitas pessoas preenchem o formulário"

**SMART:**
```
Métrica: % de prospect que preenchem Fase 1 E são aprovados para Fase 2
Target: ≥70% dos que acessam formulário completam E ≥70% são aprovados
Temporal: Medido ao dia 6 (aprovação + notificação)
Relevante: Indica que marketing traz fit + metodologia qualifica bem
Alcançável: Sim (formulário + score são o filtro)
```

---

#### **Métrica #2: Indicador Crítico Evolui**

**Antes (SOFT):** "3 clientes com resultado em 7 dias"

**SMART:**
```
Métrica: % de clientes cujo indicador crítico cresce ≥15% entre dia 1 e dia 7
Target: ≥80% dos clientes que executam blueprint 7D
Temporal: Medido dia 7 (check-in final do blueprint)
Relevante: Valida que premissa está correta + método funciona
Alcançável: Sim (indicador é cliente escolher o que mede)
Mensurável: Número/percentual/qualitativo claro (ex: "margem 20%→23%")
```

---

#### **Métrica #3: Case Studies Documentados**

**Antes (SOFT):** "3 depoimentos"

**SMART:**
```
Métrica: # de case studies com: Nome | Problema | Resultado Dia 7 | ROI | Depoimento escrito
Target: 3 case studies completos até 31/03
Temporal: Finalizado dia 23 (após cliente validar indicador)
Relevante: Evidência para marketing/confiança futura
Alcançável: Sim (você já tem os dados)
Mensurável: 0/1/2/3 case studies
```

---

#### **Métrica #4: Taxa de Conversão MVP→90D**

**Antes (SOFT):** "Oferecer acompanhamento"

**SMART:**
```
Métrica: % de clientes que completam MVP E assinam acompanhamento 90D
Target: ≥60% dos que veem indicador evoluir (dia 15)
Temporal: Medido dia 16 (após upsell #2 + decisão)
Relevante: Faturamento + validação de stickiness
Alcançável: Sim (acompanhamento é upsell natural)
Mensurável: 1/3 aprovados → 2/3 contratam 90D = 66%
```

---

### **90D — Acompanhamento**

#### **Métrica #5: Taxa de Conformidade Semanal**

**Antes (SOFT):** "Check-ins acontecem"

**SMART:**
```
Métrica: % de semanas que cliente envia check-in
Target: ≥90% das 13 semanas
Temporal: Rastreado semana a semana
Relevante: Indica engajamento do cliente
Alcançável: Sim (você acompanha)
Mensurável: 13 check-ins esperados, X check-ins recebidos = (X/13)*100%
Exemplo: 12/13 = 92% ✓
```

---

#### **Métrica #6: Crescimento do Indicador Crítico (90 dias)**

**Antes (SOFT):** "Cliente evolui"

**SMART:**
```
Métrica: % de crescimento do indicador crítico entre semana 1 e semana 13
Target: ≥20% crescimento para 90% dos clientes
Temporal: Medido semana 13 (final acompanhamento)
Relevante: Valida que escalação funciona
Alcançável: Sim (cliente já escolheu indicador)
Mensurável: (Valor Semana 13 - Valor Semana 1) / Valor Semana 1 × 100
Exemplo: Margem 20% semana 1 → 24% semana 13 = +20% ✓
```

---

#### **Métrica #7: Taxa de Retenção 90D**

**Antes (SOFT):** "Cliente não abandona"

**SMART:**
```
Métrica: % de clientes que completam 90 dias sem cancelamento
Target: ≥95% (churn máximo 5%)
Temporal: Medido dia 90
Relevante: Indica satisfação + valor entregue
Alcançável: Sim (90D é contract explícito)
Mensurável: # clientes que chegam ao dia 90 / # clientes que começaram
```

---

#### **Métrica #8: ROI por Cliente**

**Antes (SOFT):** "Resultado documentado"

**SMART:**
```
Métrica: Razão (Receita Gerada ou Custos Economizados) / (Investimento MVP + 90D)
Target: ≥3x ROI para ≥80% dos clientes
Temporal: Medido dia 90 (relatório final)
Relevante: Justifica preço da oferta
Alcançável: Sim (cliente tem dados)
Mensurável:
  Exemplo: Investimento R$ 7.000, Economia de R$ 21.000 = 3x ✓
           Investimento R$ 7.000, Receita +R$ 15.000 = 2.1x ⚠️ (abaixo target)
```

## 5. IMPACTO SE REMOVER FEATURES

### **MUST Features (Impacto = Oferta Quebra)**

```
❌ Se remover: Formulário 28Q
   → Cliente não se qualifica
   → Você não tem dados suficientes
   → Sessão vira genérica

❌ Se remover: Sessão 1:1
   → Diagnóstico não é validado com cliente
   → Premissa pode estar errada
   → Cliente não se compromete

❌ Se remover: Blueprint 7D
   → Cliente não sabe o que fazer depois
   → Não consegue validar em 7 dias
   → Valor não é tangibilizado

❌ Se remover: Caderno Inteligente (Interativo)
   → Cliente depende 100% de você pra avançar
   → Você vira gargalo
   → Acompanhamento 90D não funciona
```

---

### **SHOULD Features (Impacto = Experiência Piora, Mas Funciona)**

```
⚠️ Se remover: Diagnóstico PDF
   → Entregável é só digital (Google Doc)
   → Cliente pode imprimir do doc, mas não é professional
   → Marketing fica mais fraco (não pode compartilhar PDF)
   → Solução: Deixar como PDF depois

⚠️ Se remover: Email Automático
   → Você tem que avisar cliente manualmente
   → Demora mais, erro humano possível
   → Experiência fica mais fraca
   → Solução: Implementar automation depois
```

## 7. DEFINIÇÃO FINAL DE "PRONTO"

### **Diagnóstico é PRONTO quando:**

```
✅ Cliente preencheu formulário completo (28 questões)
✅ Você calculou score (4 dimensões, threshold validado)
✅ Sessão Fase 2 aconteceu (75–90 min, notas tiradas)
✅ Verdade Núcleo + 3 Cortes + 3 Decisões definidos (cliente validou)
✅ Blueprint 7D criado (7 páginas, específico, executável)
✅ Caderno inteligente compartilhado (cliente consegue acessar + preencher)
✅ Diagnóstico PDF formal entregue (4–5 páginas, professional)
✅ Cliente sabe as próximas ações (execução 7 dias)
```

---

### **Acompanhamento 90D é PRONTO quando:**

```
✅ Cliente enviou 13 check-ins (1 por semana)
✅ Você enviou 13 feedbacks (dentro 24h de cada check-in)
✅ 4 sessões aconteceram agendadas (notas documentadas)
✅ Indicador crítico foi mensurado (semana 1 vs semana 13)
✅ Relatório ROI foi enviado (com números verificáveis)
✅ Próximos passos foram oferecidos (renovar / escalar / referências / encerrar)
✅ Cliente teve decisão documentada (qual caminho escolheu)
```
