## Objetivo

Criar landing page que convence prospect a preencher o formulário de 28 questões. O formulário **é** o pitch — cada pergunta comunica valor, filtra, e qualifica.

## 1. HERO SECTION

### **Headline (Principal)**

```
"Você Não Falhou Porque Contratou Errado.
Você Falhou Porque Ninguém Ensinou Você a Ter Clareza."
```

**Por que funciona:**
- Específico (não é genérico)
- Validador (reafirma que o cliente não é incompetente)
- Promete solução (clareza)
- Emocional + racional

---

### **Subheadline (Secundário)**

```
"Descubra a PREMISSA ESTRUTURAL que está drenando seu negócio
— e teste a correção em 7 dias."
```

**Por que funciona:**
- Promessa clara (descobrir + testar)
- Timeline (7 dias, não meses)
- Conceito diferenciado (premissa, não sintoma)

---

### **CTA Primary (Botão Hero)**

```

```

**Variante mais agressiva:**
```

```

## 8. FAQ SECTION

### **Pergunta #1: "Quanto tempo leva do meu lado?"**

```
Total: ~2 horas na primeira semana (formulário 30 min + sessão 90 min).
Depois: 7 dias de execução conforme blueprint.

Não é consultoria contínua — é diagnóstico + validação focada.
```

### **Pergunta #2: "Qual é a garantia que vai funcionar?"**

```
Não é garantia de crescimento imediato.
É garantia de clareza.

Em 7 dias você vai SABER se a premissa está certa.
Se estiver, você escala com confiança.
Se não estiver, você ajusta antes de investir mais.

É validação, não promessa mágica.
```

### **Pergunta #3: "E depois dos 15 dias, o que acontece?"**

```
Você tem 2 caminhos:

1) Continuar sozinho com o diagnóstico + blueprint
2) Contratar acompanhamento 90D (check-ins semanais + sessões mensais)

Ambos fazem sentido. Você decide.
```

### **Pergunta #4: "Qual é meu faturamento ideal?"**

```
R$ 10k–50k/mês é o ótimo.

Mas também atendo:
- Empregados técnicos querendo escalar negócio próprio
- Prestadores de serviço vivendo de retrabalho

Se você sente "tenho algo estrutural errado", provavelmente cabe.
```

## 9. O FORMULÁRIO COMO PITCH

### **Contexto Importante**

O formulário de 28 questões **NÃO É** apenas coleta de dados.

**É pitch silencioso.**

Cada questão comunica valor:

```
"Quanto é sua margem média?"
→ Comunica: "A gente vai otimizar margem, não volume"

"Você muda de estratégia com frequência?"
→ Comunica: "Vamos trazer consistência"

"Está disposto a eliminar iniciativas?"
→ Comunica: "Isso vai ser difícil, mas necessário"

"Estima quanto perdeu nos últimos 12 meses com ajustes de rota?"
→ Comunica: "Esse número importa. Vamos evitar isso daqui pra frente"
```

### **Cada Bloco Comunica**

```
BLOCO 1 (Estrutura): "Entendo seu faturamento e operação"
BLOCO 2 (Prioridade): "Entendo seu caos de decisões"
BLOCO 3 (Coerência): "Entendo seu desalinhamento"
BLOCO 4 (Financeiro): "Entendo seu vazamento"
BLOCO 5 (Padrão): "Entendo seu loop repetido"
BLOCO 6 (Maturidade): "Entendo se você está pronto pra mudança"
```

**Resultado:** Cliente preenche e sai pensando "Wow, ela REALMENTE entende do meu problema."

## 13. DESIGN NOTES (Para Developer/Designer)

### **Layout**
- Single page (não multipágina)
- Scroll natural (não forçar lateral)
- Mobile-first (formulário precisa ser mobile-friendly)

### **Cores**
- Hero: Verde escuro + branco (confiança + clareza)
- CTA: Laranja ou verde vibrante (destaque)
- Depoimentos: Cinza claro (contraste)

### **Typography**
- Headline: 48–56px (bold)
- Subheadline: 24–28px (medium)
- Body: 16–18px (regular)
- CTA: 16–18px (bold)

### **Spacing**
- Seções: 80–100px gap (respiro)
- Elementos internos: 20–30px

### **Formulário CTA**
- Botão deve estar visível sem scroll (primeira dobra)
- Botão novamente antes de FAQ
- Botão novamente no footer (sticky se possível)
